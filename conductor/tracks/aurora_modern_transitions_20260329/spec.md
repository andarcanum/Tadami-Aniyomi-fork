# Track Specification: Aurora Modern Transitions Refinement

**1. Overview**
Улучшение «Современного» (Modern) режима анимации переходов для Aurora UI. Цель — добавить интерфейсу пространственную глубину и четкое понимание направления (Push/Pop), не перегружая его лишними движениями.

**2. Functional Requirements**
- **Directional Support:**
    - **Push (Вперед):** Экран плавно увеличивается (0.95f -> 1.0f) и проявляется (FadeIn), одновременно слегка смещаясь по горизонтали (на ~30dp).
    - **Pop (Назад):** Экран плавно уменьшается (1.0f -> 0.95f) и исчезает (FadeOut), уходя «вглубь».
- **Easing:** Замена стандартной LinearOutSlowInEasing на более премиальную CubicBezier(0.4, 0.0, 0.2, 1.0).
- **Scope:**
    - **Main Navigator:** Обновление modernSharedAxisX в Navigator.kt.
    - **HomeScreen Tabs:** Синхронизация логики переходов вкладок в HomeScreen.kt для консистентности.
- **Duration:** Сохранение текущих таймингов (300ms для входа, 170ms для выхода).

**3. Non-Functional Requirements**
- **Legacy Mode Integrity:** Режим «Классический» (Legacy/SharedAxis) должен остаться без изменений.
- **Performance:** Анимации должны оставаться плавными (60+ FPS) на устройствах среднего сегмента.

**4. Acceptance Criteria**
- [ ] Переходы в Современном режиме визуально отличаются при движении «туда» и «обратно».
- [ ] Анимация вкладок на главном экране соответствует новой стилистике.
- [ ] Переключение в настройках на «Классический» режим возвращает старое поведение.
